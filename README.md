# OutlineView

`OutlineView` is a small SwiftUI outline/tree view for macOS and iOS. It is designed for app-specific hierarchical models such as file browsers, layer lists, project navigators, bookmark trees, note trees, and similar structured data.

The control owns presentation behavior that is awkward to express with `List(_:children:)`: indentation, disclosure controls, selection highlighting, row-relative drag targets, root drops, and explicit move callbacks. Your app owns the model, validation, persistence, and actual mutation.

## Requirements

- Swift 6
- macOS 13 or later
- iOS 16 or later

## Data Model

Rows must be `Identifiable`. Children are supplied through a key path to an optional collection:

```swift
struct FileNode: Identifiable {
	let id: URL
	var name: String
	var children: [FileNode]?
}
```

`children == nil` means the row is a leaf, such as a file. `children != nil` means the row is a branch, such as a folder. Empty branches should use `[]`, not `nil`, so they remain expandable and valid "drop inside" destinations.

## Basic Usage

```swift
struct Sidebar: View {
	@State private var nodes: [FileNode]
	@State private var selection: Set<URL> = []
	@State private var expanded: Set<URL> = []

	var body: some View {
		OutlineView(
			nodes,
			children: \.children,
			selection: $selection,
			expanded: $expanded
		) { node in
			Label(node.name, systemImage: node.children == nil ? "doc" : "folder")
		}
	}
}
```

`selection` and `expanded` are caller-owned bindings. The current implementation supports single-click replacement selection. Multi-selection gestures and keyboard navigation are future work.

## Moves and Drops

Use the `onMove` initializer for generic tree operations:

```swift
OutlineView(
	nodes,
	children: \.children,
	selection: $selection,
	expanded: $expanded,
	onMove: handleMove
) { node in
	Text(node.name)
}

func handleMove(_ move: OutlineMove<URL>) -> Bool {
	switch move.destination {
	case .root:
		// Move source to the root level.
		return true
	case let .before(target):
		// Move source before target among target's siblings.
		return true
	case let .inside(target):
		// Move source inside target.
		return true
	case let .after(target):
		// Move source after target among target's siblings.
		return true
	}
}
```

The callback reports intent only. Return `true` after accepting and mutating your model. Return `false` to reject the drop. For directory browsers, this is where you would prevent moving a directory into its own subtree, resolve name conflicts, check permissions, and call your file-operation layer.

The older `onDrop: (OutlineDrop<ID>) -> Bool` initializer is still available for row-relative drops. It cannot represent root drops, so new code should prefer `onMove`.

## Directory Browser Fit

`OutlineView` can present a file-system tree as long as the caller adapts directories to branches and files to leaves:

- Directory with loaded children: `children = [...]`
- Empty directory: `children = []`
- File: `children = nil`

The view does not perform file I/O. A directory browser should keep file operations outside the control and use `OutlineMove` as a UI-level command.

## Example

`Examples/MemoTree` is a simple macOS/iOS harness generated by XcodeGen. It demonstrates:

- Folders as branches and memos as leaves
- Empty folder drops
- Root-level moves
- Rejection of self-drops and moves into a node's own subtree
- A small drop log for smoke testing

Open `OutlineView.xcworkspace`, or build the example directly:

```sh
xcodebuild build -project Examples/MemoTree/MemoTree.xcodeproj -scheme MemoTree -destination 'platform=macOS'
xcodebuild build -project Examples/MemoTree/MemoTree.xcodeproj -scheme MemoTree -destination 'generic/platform=iOS Simulator'
```

## Testing

Run package tests from the repository root:

```sh
swift test
```

The current tests cover tree flattening, empty branch behavior, drop-zone math, non-Codable IDs, and `OutlineMove` compatibility with the legacy `OutlineDrop` API.

## Current Scope

Implemented:

- Generic tree rendering from caller-owned data
- Caller-owned selection and expansion state
- Empty branch detection
- Row-relative drag destinations
- Root drop destination through `onMove`
- Non-Codable row IDs
- Caller-supplied row context menus that remain reachable when drop zones are
  enabled

Not yet implemented:

- Multi-selection
- Keyboard navigation
- Rename/edit flows
- Lazy loading hooks
- External file drops
- Accessibility polish
- Large-tree virtualization beyond `LazyVStack`
